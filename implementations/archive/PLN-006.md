# PLN-006 — Organizzazione prompt Models Agent secondo pattern F009

## Contesto

Il pattern F009 (Prompt Organization Architecture) definisce una metodologia standardizzata per organizzare i testi dei prompt LLM a livello di agente.

Finora sono stati implementati quattro agenti:
- ✅ **Supervisor Agent** — `supervisor_agent_prompts.py` (PLN-003)
- ✅ **Request Parser Agent** — `request_parser_prompts.py` (PLN-004)
- ✅ **Final Responder Agent** — `final_responder_prompts.py` (PLN-004)
- ✅ **SaferCast Agent** — `safercast_agent_prompts.py` (PLN-005)

Il **Models Agent** (Simulations) utilizza attualmente una classe `ModelsPrompts` nel file `models_agent.py` senza seguire il pattern F009. 

Questo piano estende il pattern F009 al Models Agent, creando un modulo dedicato `models_agent_prompts.py` che organizza i prompt con la stessa struttura gerarchica usata dal Supervisor e dal SaferCast Agent.

## §1 — Analisi costrutti attuali

Nel file `models_agent.py` esiste una classe `ModelsPrompts` con i seguenti artefatti:

| Artefatto | Metodo/Attributo | Input | Output | Contesto |
|---|---|---|---|---|
| Instruzioni di sistema | `TOOL_SELECTION_SYSTEM` | — | `str` | System message per tool selection |
| Prompt iniziale | `initial_request(state)` | `MABaseGraphState` | `str` | Human message con goal + parsed_request + relevant_layers |
| Prompt reinvocazione | `reinvocation_request(state)` | `MABaseGraphState` | `str` | Human message dopo rifiuto: include current invocation + user feedback |

Questi si mappano naturalmente al pattern F009:

```
ModelsPrompts
├── MainContext.stable() → istruzioni di sistema (tool selection)
├── ToolSelection.*
│   ├── InitialRequest.stable(state)
│   └── ReinvocationRequest.stable(state)
└── v001() variants per A/B testing
```

## §2 — Implementazione ModelsPrompts

La classe `ModelsPrompts` segue la struttura standard F009 (identica a `SaferCastPrompts` da PLN-005):

### Gerarchia logica

```
ModelsPrompts (classe principale)
├── MainContext
│   ├── stable() → Prompt("System role for tool selection")
│   └── v001() → alternative version
│
└── ToolSelection
    ├── InitialRequest
    │   ├── stable(state) → Prompt(goal + parsed_request + layers)
    │   └── v001(state) → alternative
    │
    └── ReinvocationRequest
        ├── stable(state) → Prompt(reinvocation feedback loop)
        └── v001(state) → alternative
```

### Signature dei metodi

| Metodo | Firma | Ritorna |
|---|---|---|
| `MainContext.stable()` | `() -> Prompt` | Istruzioni di sistema statiche |
| `ToolSelection.InitialRequest.stable()` | `(state: MABaseGraphState, **kwargs) -> Prompt` | Prompt iniziale context-aware |
| `ToolSelection.ReinvocationRequest.stable()` | `(state: MABaseGraphState, **kwargs) -> Prompt` | Prompt reinvocazione context-aware |

Ogni versione `v###()` ha la stessa firma della corrispondente `stable()`.

### Dataclass Prompt

Ogni metodo ritorna un oggetto `Prompt` con i campi:

```python
Prompt({
    "title": str,              # Nome mnemonico (es. "SimulationToolSelectionContext")
    "description": str,        # Descrizione breve
    "command": str,            # Flag speciale (solitamente "")
    "message": str,            # Testo completo del prompt
})
```

## §3 — Migrazione da ModelsPrompts

Il file `models_agent.py` verrà aggiornato secondo il pattern:

### Prima (legacy)

```python
class ModelsPrompts:
    TOOL_SELECTION_SYSTEM = "You are a specialized simulations agent..."
    
    @staticmethod
    def initial_request(state: MABaseGraphState) -> str:
        # costruisce il messaggio con goal + parsed_request
        return f"Goal: {goal}..."

# Utilizzo
messages = [
    SystemMessage(content=ModelsPrompts.TOOL_SELECTION_SYSTEM),
    HumanMessage(content=ModelsPrompts.initial_request(state))
]
```

### Dopo (F009)

```python
from ..prompts.models_agent_prompts import ModelsPrompts

# Utilizzo
system_prompt = ModelsPrompts.MainContext.stable()
human_prompt = ModelsPrompts.ToolSelection.InitialRequest.stable(state)

messages = [
    system_prompt.to(SystemMessage),
    human_prompt.to(HumanMessage)
]
```

Il metodo `Prompt.to(MessageClass)` converte l'oggetto `Prompt` in un `BaseMessage` di LangChain.

### Punti di modifica in `models_agent.py`

1. Rimuovere la classe `ModelsPrompts` (non più necessaria)
2. Importare `ModelsPrompts` da `ma.prompts.models_agent_prompts`
3. In `_build_invocation_messages()`, usare `ModelsPrompts.stable()` e `Prompt.to()`
4. Adattare i nomi di costante se necessario (es. `STATE_MODELS_*` rimangono uguali)

## Note e decisioni

1. **Nessuna funzionalità cambia**: il refactor preserva il comportamento di input/output; è puramente organizzativo.

2. **Coerenza con PLN-005**: Questo piano segue esattamente il pattern di PLN-005 (SaferCast Agent), assicurando coerenza architetturale.

3. **Versionamento dei prompt**: I metodi `v###()` sono placeholder per test futuro e A/B testing. Per ora, tutti i prompt useranno `stable()`.

4. **Importazione dei Cost**: Il modulo `models_agent_prompts.py` importa costanti da `models_agent.py` (es. `MODELS_AGENT_DESCRIPTION`). Tenere separate le sezioni "metadata" dalle "prompts" per chiarezza.

5. **Documentazione**: La docstring di `ModelsPrompts` e di ogni classe/metodo deve essere chiara e seguire la convenzione in `supervisor_agent_prompts.py` e `safercast_agent_prompts.py`.
