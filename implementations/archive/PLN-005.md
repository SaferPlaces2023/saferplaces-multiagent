# PLN-005 — Organizzazione prompt SaferCast Agent secondo pattern F009

## Contesto

Il pattern F009 (Prompt Organization Architecture) definisce una metodologia standardizzata per organizzare i testi dei prompt LLM a livello di agente.

Finora sono stati implementati tre agenti:
- ✅ **Supervisor Agent** — `supervisor_agent_prompts.py` (PLN-003)
- ✅ **Request Parser Agent** — `request_parser_prompts.py` (PLN-004)
- ✅ **Final Responder Agent** — `final_responder_prompts.py` (PLN-004)

Il **SaferCast Agent** (Data Retriever) utilizza attualmente una classe `RetrieverPrompts` nel file `safercast_agent.py` senza seguire il pattern F009. 

Questo piano estende il pattern F009 al SaferCast Agent, creando un modulo dedico `safercast_agent_prompts.py` che organizza i prompt con la stessa struttura gerarchica usata dal Supervisor.

## §1 — Analisi costrutti attuali

Nel file `safercast_agent.py` esiste una classe `RetrieverPrompts` con i seguenti artefatti:

| Artefatto | Metodo/Attributo | Input | Output | Contexto |
|---|---|---|---|---|
| Instruzioni di sistema | `TOOL_SELECTION_SYSTEM` | — | `str` | System message per tool selection |
| Prompt iniziale | `initial_request(state)` | `MABaseGraphState` | `str` | Human message con goal + parsed_request + relevant_layers |
| Prompt reinvocazione | `reinvocation_request(state)` | `MABaseGraphState` | `str` | Human message dopo rifiuto: include current invocation + user feedback |

Questi si mappano naturalmente al pattern F009:

```
SaferCastPrompts
├── MainContext.stable() → istruzioni di sistema (tool selection)
├── ToolSelection.*
│   ├── InitialRequest.stable(state)
│   └── ReinvocationRequest.stable(state)
└── v001() variants per A/B testing
```

## §2 — Implementazione SaferCastPrompts

La classe `SaferCastPrompts` segue la struttura standard F009:

### Gerbarchia logica

```
SaferCastPrompts (classe principale)
├── MainContext
│   ├── stable() → Prompt("System role for tool selection")
│   └── v001() → alternative version
│
└── ToolSelection
    ├── InitialRequest
    │   ├── stable(state) → Prompt(goal + parsed_request + layers)
    │   └── v001(state) → alternative
    │
    └── ReinvocationRequest
        ├── stable(state) → Prompt(reinvocation feedback loop)
        └── v001(state) → alternative
```

### Signature dei metodi

| Metodo | Firma | Ritorna |
|---|---|---|
| `MainContext.stable()` | `() -> Prompt` | Istruzioni di sistema statiche |
| `ToolSelection.InitialRequest.stable()` | `(state: MABaseGraphState, **kwargs) -> Prompt` | Prompt iniziale context-aware |
| `ToolSelection.ReinvocationRequest.stable()` | `(state: MABaseGraphState, **kwargs) -> Prompt` | Prompt reinvocazione context-aware |

Ogni versione `v###()` ha la stessa firma della corrispondente `stable()`.

### Dataclass Prompt

Ogni metodo ritorna un oggetto `Prompt` con i campi:

```python
Prompt({
    "title": str,              # Nome mnemonico (es. "ToolSelectionContext")
    "description": str,        # Descrizione breve
    "command": str,            # Flag speciale (solitamente "")
    "message": str,            # Testo completo del prompt
})
```

## §3 — Migrazione da RetrieverPrompts

Il file `safercast_agent.py` verrà aggiornato secondo il pattern:

### Prima (legacy)

```python
class RetrieverPrompts:
    TOOL_SELECTION_SYSTEM = "You are a specialized agent..."
    
    @staticmethod
    def initial_request(state: MABaseGraphState) -> str:
        # costruisce il messaggio con goal + parsed_request
        return f"Goal: {goal}..."

# Utilizzo
messages = [
    SystemMessage(content=RetrieverPrompts.TOOL_SELECTION_SYSTEM),
    HumanMessage(content=RetrieverPrompts.initial_request(state))
]
```

### Dopo (F009)

```python
from ..prompts.safercast_agent_prompts import SaferCastPrompts

# Utilizzo
system_prompt = SaferCastPrompts.MainContext.stable()
human_prompt = SaferCastPrompts.ToolSelection.InitialRequest.stable(state)

messages = [
    system_prompt.to(SystemMessage),
    human_prompt.to(HumanMessage)
]
```

Il metodo `Prompt.to(MessageClass)` converte l'oggetto `Prompt` in un `BaseMessage` di LangChain:

```python
prompt = SaferCastPrompts.MainContext.stable()
message = prompt.to(SystemMessage)  # → SystemMessage(content=prompt.message)
```

### Punti di modifica in `safercast_agent.py`

1. Rimuovere la classe `RetrieverPrompts` (non più necessaria)
2. Importare `SaferCastPrompts` da `ma.prompts.safercast_agent_prompts`
3. In `_build_invocation_messages()`, usare `SaferCastPrompts.safe()` e `Prompt.to()`
4. Adattare i nomi di costante se necessario (es. `STATE_RETRIEVER_*` rimangono uguali)

def _override_initial_request(state):
    return Prompt({"title": "...", "description": "...", "message": "_TEST_"})

with patch.object(SaferCastPrompts.ToolSelection.InitialRequest, "stable", _override_initial_request):
    # Esegui test; il prompt sarà sovrascitto a runtime

## Note e decisioni

1. **Nessuna funzionalità cambia**: il refactor preserva il comportamento di input/output; è puramente organizzativo.

2. **Compatibilità verso `ModelAgent`**: Questo piano prepara la strada per PLN-006 (Models Agent). La stessa struttura verrà usata lì.

3. **Versionamento dei prompt**: I metodi `v###()` sono placer per test futuro e A/B testing. Per ora, tutti i prompt useranno `stable()`.

4. **Importazione dei Cost**: Il modulo `safercast_agent_prompts.py` importa costanti da `safercast_agent.py` (es. `SAFERCAST_AGENT_DESCRIPTION`). Tenere separate le sezioni "metadata" dalle "prompts" per chiarezza.

5. **Documentazione**: La docstring di `SaferCastPrompts` e di ogni classe/metodo deve essere chiara e seguire la convenzione in `supervisor_agent_prompts.py`.
